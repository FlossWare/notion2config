# notion-dnsmasq.sh - Detailed Documentation

Generate dnsmasq configuration from Notion database.

## Overview

`notion-dnsmasq.sh` queries a Notion database containing network device inventory (hostnames, IPs, MAC addresses) and generates dnsmasq configuration with:

1. **Static DHCP leases** (`dhcp-host`) - For devices with both MAC and IP
2. **DNS A records** (`address`) - For devices with IP only

This enables infrastructure-as-code where your network device inventory lives in Notion and dnsmasq configuration is automatically generated.

## Usage

### Basic Syntax

```bash
notion-dnsmasq.sh [OPTIONS]
```

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `--output FILE`, `-o FILE` | Write output to FILE | stdout |
| `--dry-run`, `-n` | Preview output without writing | false |
| `--token TOKEN`, `-t TOKEN` | Notion integration token | `$NOTION_TOKEN` |
| `--database ID`, `-d ID` | Notion database ID | hardcoded |
| `--help`, `-h` | Show help message | - |

### Environment Variables

| Variable | Description | Required |
|----------|-------------|----------|
| `NOTION_TOKEN` | Notion integration token | Yes |

## Examples

### Preview Output (Dry Run)

```bash
export NOTION_TOKEN=secret_xxx
./notion-dnsmasq.sh --dry-run
```

### Generate to File

```bash
./notion-dnsmasq.sh --output /tmp/dnsmasq-hosts.conf
```

### Use Different Database

```bash
./notion-dnsmasq.sh --database 123456789abc -o hosts.conf
```

### Direct to Production (with sudo)

```bash
sudo -E ./notion-dnsmasq.sh -o /etc/dnsmasq.d/hosts.conf
```

## Database Requirements

### Required Columns

The Notion database must have these columns:

#### Name (Title)
- **Type:** Title
- **Purpose:** Hostname for DNS/DHCP
- **Format:** Alphanumeric, hyphens, underscores
- **Examples:** `server-01`, `workstation-jane`, `iot-sensor-3`

**Processing:**
- Converted to lowercase
- Spaces replaced with hyphens
- Used as hostname in both dhcp-host and address entries

#### IP (Text)
- **Type:** Text (or Rich Text)
- **Purpose:** IPv4 address
- **Format:** `x.x.x.x` (standard dotted notation)
- **Examples:** `192.168.1.100`, `10.0.0.50`

**Processing:**
- Used as-is (no validation)
- Required for both dhcp-host and address entries

#### MAC (Text)
- **Type:** Text (or Rich Text)
- **Purpose:** MAC address for DHCP binding
- **Format:** Flexible (see below)
- **Examples:** `AA:BB:CC:DD:EE:FF`, `aa-bb-cc-dd-ee-ff`, `aabbccddeeff`

**Processing:**
- Converted to lowercase
- Normalized to colon-separated format
- Non-hexadecimal characters removed
- Optional (only needed for DHCP static leases)

### Optional Columns

These columns can exist but are ignored:

- **Location** (Select) - Physical location
- **Type** (Select) - Device type (server, workstation, etc.)
- **OS** (Select) - Operating system
- **Status** (Select) - Active, inactive, maintenance
- **Notes** (Text) - Comments
- **Tags** (Multi-select) - Arbitrary tags

## Output Format

### File Header

```
# dnsmasq.conf — auto-generated by notion-dnsmasq.sh
# Source : Notion Compute database (30b3d6434d24819dbc06e0046b140c30)
# Generated: 2026-05-21T15:30:00Z
# DO NOT EDIT BY HAND — regenerate with notion-dnsmasq.sh
```

### DHCP Host Entries

For devices with **both MAC and IP**:

```
# ── Static DHCP leases ────────────────────────────────────────────────────────
# dhcp-host=<mac>,<hostname>,<ip>

dhcp-host=aa:bb:cc:dd:ee:ff,server-01,192.168.1.100
dhcp-host=11:22:33:44:55:66,workstation-jane,192.168.1.101
```

### DNS Address Entries

For **all devices with IP** (including those with DHCP entries):

```
# ── DNS address records ──────────────────────────────────────────────────────
# address=/<hostname>/<ip>

address=/server-01/192.168.1.100
address=/workstation-jane/192.168.1.101
address=/cloud-server/203.0.113.50
```

**Note:** Devices with static DHCP also get DNS entries so they resolve even when DHCP is off.

## Processing Logic

### Decision Tree

For each database entry:

```
┌─ Entry from Notion
│
├─ Has NAME?
│  ├─ NO  → Skip (invalid entry)
│  └─ YES → Continue
│
├─ Has MAC AND IP?
│  ├─ YES → Generate dhcp-host + address
│  └─ NO  → Check IP only
│
└─ Has IP (no MAC)?
   ├─ YES → Generate address only
   └─ NO  → Skip (no IP)
```

### MAC Address Normalization

Input formats accepted:
```
AA:BB:CC:DD:EE:FF    → aa:bb:cc:dd:ee:ff
aa-bb-cc-dd-ee-ff    → aa:bb:cc:dd:ee:ff
aabbccddeeff         → aa:bb:cc:dd:ee:ff
AA-BB-CC-DD-EE-FF    → aa:bb:cc:dd:ee:ff
```

Invalid characters are stripped:
```
aa:bb:cc:dd:ee:ff (notes)  → aa:bb:cc:dd:ee:ff
```

### Hostname Normalization

```
Server 01         → server-01
Workstation_Jane  → workstation_jane
IOT Sensor #3     → iot-sensor-3
```

## Behavior

### Pagination

Automatically handles large databases:
- Fetches 100 entries per request
- Follows `next_cursor` for additional pages
- No size limit

### Backups

Before overwriting existing config:
```bash
cp /etc/dnsmasq.d/hosts.conf /etc/dnsmasq.d/hosts.conf.bak
```

### Service Reload

If systemd and dnsmasq are active:
```bash
systemctl reload dnsmasq
```

On FreeBSD:
```bash
service dnsmasq reload
```

### Error Handling

**Notion API errors:**
- Invalid token → Exit with error message
- Database not found → Exit with error message
- Network timeout → Curl error displayed

**Invalid data:**
- Missing name → Entry skipped (logged to stderr)
- Missing IP/MAC → Entry skipped (logged to stderr)
- Invalid MAC → Normalized (non-hex chars removed)

## Integration Examples

### Systemd Service

Create `/etc/systemd/system/notion-dnsmasq.service`:

```ini
[Unit]
Description=Update dnsmasq from Notion
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
EnvironmentFile=/etc/notion-infrastructure/token.env
ExecStart=/opt/notion-infrastructure/converters/notion-dnsmasq.sh -o /etc/dnsmasq.d/hosts.conf
User=root

[Install]
WantedBy=multi-user.target
```

### Systemd Timer

Create `/etc/systemd/system/notion-dnsmasq.timer`:

```ini
[Unit]
Description=Update dnsmasq from Notion every hour
Requires=notion-dnsmasq.service

[Timer]
OnBootSec=5min
OnUnitActiveSec=1h
Persistent=true

[Install]
WantedBy=timers.target
```

Enable:
```bash
sudo systemctl enable --now notion-dnsmasq.timer
```

### Cron Job

```bash
# /etc/cron.d/notion-dnsmasq
# Update every hour at 5 minutes past
5 * * * * root NOTION_TOKEN=secret_xxx /opt/notion-infrastructure/converters/notion-dnsmasq.sh -o /etc/dnsmasq.d/hosts.conf 2>&1 | logger -t notion-dnsmasq
```

### Git Tracking

Track changes to generated config:

```bash
#!/bin/bash
# /opt/notion-infrastructure/update-and-commit.sh

cd /etc/dnsmasq.d

# Generate new config
NOTION_TOKEN=secret_xxx /opt/notion-infrastructure/converters/notion-dnsmasq.sh -o hosts.conf

# Commit if changed
if ! git diff --quiet hosts.conf; then
    git add hosts.conf
    git commit -m "Update from Notion: $(date -I)"
    git push
fi
```

## Troubleshooting

### Issue: "NOTION_TOKEN is not set"

**Cause:** Environment variable not exported

**Fix:**
```bash
export NOTION_TOKEN=secret_xxx
./notion-dnsmasq.sh --dry-run
```

Or pass via argument:
```bash
./notion-dnsmasq.sh --token secret_xxx --dry-run
```

### Issue: "Notion API returned an error"

**Cause:** Token invalid or database not shared

**Debug:**
```bash
# Test token
curl -H "Authorization: Bearer $NOTION_TOKEN" \
     -H "Notion-Version: 2022-06-28" \
     https://api.notion.com/v1/users/me

# Test database access
curl -X POST \
     -H "Authorization: Bearer $NOTION_TOKEN" \
     -H "Notion-Version: 2022-06-28" \
     -H "Content-Type: application/json" \
     -d '{"page_size": 1}' \
     https://api.notion.com/v1/databases/YOUR_DB_ID/query
```

### Issue: "dnsmasq reload failed"

**Cause:** Syntax error in generated config

**Debug:**
```bash
# Test dnsmasq config
sudo dnsmasq --test -C /etc/dnsmasq.conf

# Check generated file
cat /etc/dnsmasq.d/hosts.conf

# Check dnsmasq logs
sudo journalctl -u dnsmasq -n 50
```

### Issue: Entries being skipped

**Cause:** Missing required fields in Notion

**Check stderr output:**
```bash
./notion-dnsmasq.sh -o hosts.conf 2>&1 | grep SKIP
```

Example output:
```
# SKIP: old-server — no IP or MAC recorded
# SKIP: test-device — no IP or MAC recorded
```

**Fix:** Add IP (and optionally MAC) to those entries in Notion.

## Security Best Practices

1. **Protect the token:**
   - Store in environment file with 600 permissions
   - Never commit to git
   - Rotate regularly

2. **Validate generated config:**
   - Always test with `--dry-run` first
   - Use `dnsmasq --test` before deployment

3. **Limit integration permissions:**
   - Notion integration should have read-only access
   - Share only the specific database needed

4. **Monitor changes:**
   - Log all config updates
   - Use git to track changes
   - Alert on unexpected modifications

## Performance

**Notion API limits:**
- 3 requests per second
- This script respects rate limits via pagination

**Database size:**
- Tested with 1000+ entries
- No practical limit (pagination handles any size)

**Generation time:**
- ~1-2 seconds for 100 entries
- ~5-10 seconds for 1000 entries
