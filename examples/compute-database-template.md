# Notion Compute Database Template

Use this template to create a Notion database for managing network device inventory that works with `notion-dnsmasq.sh`.

## Quick Setup

1. **Duplicate this template** in Notion
2. **Customize columns** as needed (keep required columns)
3. **Share with integration** (see main README)
4. **Get database ID** from URL
5. **Run converter** with your token and database ID

## Required Columns

These columns **must exist** with exact names (case-sensitive):

### Name (Title)
- **Property Type:** Title
- **Description:** Hostname for the device
- **Format:** Letters, numbers, hyphens, underscores
- **Examples:**
  - `server-01`
  - `workstation-jane`
  - `router-main`
  - `iot-thermostat-bedroom`

### IP (Text or Rich Text)
- **Property Type:** Text (or Rich Text)
- **Description:** IPv4 address
- **Format:** `x.x.x.x`
- **Examples:**
  - `192.168.1.100`
  - `10.0.0.50`
  - `172.16.0.10`

### MAC (Text or Rich Text)
- **Property Type:** Text (or Rich Text)
- **Description:** MAC address (optional for DNS-only entries)
- **Format:** Any common MAC format (will be normalized)
- **Examples:**
  - `AA:BB:CC:DD:EE:FF`
  - `aa-bb-cc-dd-ee-ff`
  - `aabbccddeeff`

## Usage Tips

### Filtering

Use Notion views to filter/organize:

1. **Active Devices** - Status = "✅ Active"
2. **Servers Only** - Type = "Server"
3. **By Location** - Group by Location
4. **Need DHCP** - MAC is not empty
5. **DNS Only** - MAC is empty

### Automation

Set up automations in Notion:
- Auto-assign status when created
- Notify when device added
- Track changes with page history

### Integration

The converter queries **all pages** in the database, regardless of views/filters. Use Status column to control what's active.

### Best Practices

1. **Use consistent naming:**
   - Lowercase hostnames
   - Hyphens instead of underscores
   - Pattern: `type-location-number` (e.g., `server-dc-01`)

2. **Keep MACs updated:**
   - Essential for DHCP static leases
   - Use device labels/documentation

3. **Document changes:**
   - Use Notes column for configuration details
   - Track major changes in page history

4. **Regular audits:**
   - Mark decommissioned devices
   - Remove old entries
   - Verify IP assignments

## Advanced: Formulas

Add calculated columns:

### FQDN (Formula)
```
prop("Name") + ".local"
```
Result: `server-01.local`

### Has DHCP (Formula)
```
if(empty(prop("MAC")), "DNS Only", "DHCP + DNS")
```
Result: Shows entry type

### IP Range (Formula)
```
if(test(prop("IP"), "^192\.168\.1\."), "LAN", 
if(test(prop("IP"), "^10\.0\.0\."), "VPN", "Other"))
```
Result: Categorizes by IP range

## Troubleshooting

### Entry Being Skipped

**Problem:** Entry not in generated config

**Check:**
1. Name column has value
2. IP column has value
3. Status is not filtering it (Notion view vs. database)
4. Check stderr output: `./notion-dnsmasq.sh --dry-run 2>&1 | grep SKIP`

### Invalid MAC Address

**Problem:** MAC address format not recognized

**Solution:** Use any of these formats:
- `AA:BB:CC:DD:EE:FF` (colon-separated)
- `aa-bb-cc-dd-ee-ff` (hyphen-separated)
- `aabbccddeeff` (no separators)

### Hostname with Spaces

**Problem:** Hostname has spaces

**Result:** Automatically converted to hyphens
- Input: `Server 01`
- Output: `server-01`

## Migration from Spreadsheet

If migrating from Excel/Google Sheets:

1. Export to CSV
2. Import to Notion (File → Import → CSV)
3. Map columns:
   - Hostname → Name
   - IP Address → IP
   - MAC Address → MAC
4. Add additional columns as needed
5. Verify data and run converter
